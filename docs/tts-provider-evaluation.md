# TTS Provider 採用判定基準

genz-money チャンネルにおける日本語 TTS provider の比較評価と採用判定ルール。

## 比較対象

| Provider | 強み | 弱み |
|----------|------|------|
| ElevenLabs | 自然さ・表現力・多言語 | 日本語アクセント制御が相対的に弱め |
| VOICEVOX | 日本語の読み補正・アクセント制御・無料 | 「中品質」・日本語専用 |

## 評価観点（各 1〜5）

| 観点 | 説明 | genz-money での重み |
|------|------|-------------------|
| 誤読率 | 金融用語・制度語の読み間違い | 最重要 |
| 数字・制度語の安定性 | 「360万円」「非課税」等の読み | 最重要 |
| 明瞭さ・聞き取りやすさ | 文の区切りが明確か | 重要 |
| 間・ポーズの自然さ | 不自然な途切れがないか | 重要 |
| イントネーションの自然さ | 日本語として自然な抑揚か | 重要 |
| 信頼感・説明向きか | 金融教育に適したトーンか | 重要 |
| Shorts向けの抜け感 | 若者向けでも軽薄すぎないか | 中程度 |

## 採用判定ルール

### VOICEVOX 採用（genz-money のみ）
以下のすべてを満たす場合:
- 誤読率で VOICEVOX が明確に優位（スコア差 2 以上）
- 数字・制度語の安定性で VOICEVOX が優位
- 信頼感スコアが 3 以上

### ElevenLabs 継続
以下のいずれかに該当する場合:
- 総合スコアで大差がない（差 3 点以内）
- ElevenLabs が自然さで勝ち、誤読は辞書で潰せる見込み
- 運用統一性を優先する判断

### 併用
以下の場合:
- VOICEVOX が誤読・安定性で優位だが、ElevenLabs の自然さも捨てがたい
- 動画の種類（制度説明 vs カジュアル）で使い分ける戦略

## 比較ベンチの進め方

### 前提条件
- TTS 専用原稿（15〜25字/文）を使用すること
- 読み辞書適用済みのテキストで比較すること
- 最低でも 30 秒の同一原稿で比較すること

### 比較構成（推奨）

```bash
acs tts-bench-provider genz-money <原稿パス> \
  --elevenlabs-voices <voice1>,<voice2> \
  --voicevox-speakers <speaker1>,<speaker2> \
  --label provider-comparison-30s
```

### 評価手順
1. 生成された音声ファイルを全件聴く
2. レポートのスコアシートに 1〜5 で記入
3. メモ欄に具体的な違和感を記録
4. 採用判定ルールに基づき判断
5. 判定結果をレポートの採用判定セクションに記入

## チャンネル別の方針

| チャンネル | 現行 TTS | VOICEVOX 検討 |
|-----------|---------|--------------|
| genz-money | ElevenLabs | 比較検証中（優先） |
| japanese-mindset | ElevenLabs | 対象外（英語） |
| chill-culture | ElevenLabs | 将来検討可能 |

## VOICEVOX 利用時の注意

- クレジット表記が必要（動画概要欄に「音声: VOICEVOX」等）
- speaker ごとに商用利用規約が異なるため、採用前に確認
- VOICEVOX ENGINE がローカルで起動している必要あり
- 出力フォーマットは WAV（ElevenLabs は MP3）。動画編集時に統一が必要
