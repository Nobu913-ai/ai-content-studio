## 2. トーン・表現ルール

### 推奨トーン (warm-friendly)

- 実体験ベース
- やさしい・共感ベース
- 保存したくなる実用系
- 友達に送りたくなる
- 「個人的に」「傾向として」「初めて頼む店で」のような柔らかい表現

### 禁止トーン (店舗批判・煽り回避)

- 「地雷店を見抜け」「この店はダメ」「このジャンルは頼むな」
- 「情弱はここで損してる」「Uber で損する人の特徴」
- 「絶対」「必ず」等の強い断定
- 「外食する人は損している」「テイクアウトは手抜きではない」（防御的表現も避ける）

### 推奨フック例

- 「テイクアウトグルメ、甘く見てない？」(1本目専用・少し攻めの引き)
- 「Uber Eats で失敗したくない人、ここ見てください」
- 「デリバリーで麺類を頼んで後悔したことありませんか？」
- 「初めての店でテイクアウトする前に、これだけ見て」
- 「個人的に失敗しにくい選び方」

### 1カット1メッセージ原則

- 「お金の初期設定」と同じく、1カット1メッセージを維持
- ただし冒頭は **情報密度より感情・共感** を優先 (お金の初期設定は数字でフック、グルメは共感でフック)

## 5. 動画テンプレート (4種)

### テンプレ1: 3ポイント解説

- 尺: 25-30秒
- 用途: 店選び・口コミ・写真・配達時間
- 構成: フック → よくある失敗 → 3ポイント → 補足 → 保存CTA

### テンプレ2: 向き・不向き診断

- 尺: 20-25秒
- 用途: 料理ジャンル別
- 構成: 二択提示 → メリット → 注意点 → 推奨注文方法 → コメント誘導

### テンプレ3: あるある対処法

- 尺: 20秒
- 用途: トラブル・共感
- 構成: あるある提示 → なぜ起こるか → 次回の対策 → 保存・共感コメント誘導

### テンプレ4: 比較型

- 尺: 25-30秒
- 用途: バーガー vs カレー、ピザ vs 麺 など
- 構成: 二択提示 → A特徴 → B特徴 → 結論 → どっち派？コメント誘導

### コンセプト型 (1本目専用)

- 尺: 30秒
- 用途: ブランド世界観・期待感醸成
- 構成: フック → 魅力訴求 → 生活感シーン → シリーズ予告

## 6. 投稿戦略

### Instagram Reels が主戦場

- 既存フォロワー6,000+ ベース → 初動が確保しやすい
- ハッシュタグは **5個まで** (Instagram 2024-2025 仕様)
- カルーセル投稿との併用も検討 (保存率最大化)

### 横展開 (将来検討)

- TikTok: 同 mp4 流用可能
- YouTube Shorts: 同 mp4 流用可能
- X: 告知ポスト

ただし**初期はInstagram集中**で他プラットフォーム展開は反応を見てから判断。

### 最初の5本 (確定)

| # | テーマ | 型 |
|---|---|---|
| 1 | **テイクアウトグルメ、甘く見てない？** | コンセプト型 |
| 2 | Uber Eats で失敗しない店選び3つの見方 | 3ポイント解説 |
| 3 | 評価4.5なのに微妙な店、ここを見落としがち | 失敗回避型 |
| 4 | テイクアウト向きの料理・向かない料理 | 比較型 |
| 5 | 疲れた夜にテイクアウトがありがたい理由 | 共感型 |

#### 1本目の制作方針 (確定)

- ノウハウではなく **「リール開始宣言・世界観づくり」**
- フック「テイクアウトグルメ、甘く見てない？」 (少し攻めの引き)
- 本文トーンは柔らかく、外食を否定しない
- 最後にシリーズ予告を入れる

#### 1本目の台本 (採用版)

```text
テイクアウトグルメ、甘く見てない？

外食もいいけど、
家で食べるお店の味って、けっこう幸せです。

並ばなくていい。
周りを気にしなくていい。
好きな動画を見ながら食べてもいい。

しかも、選び方を知ってると、
テイクアウトの満足度はかなり変わります。

このリールでは、
テイクアウトグルメの楽しみ方や、
デリバリーで失敗しない選び方を紹介していきます。
```

### 投稿時間帯 (テイクアウト・デリバリー文脈)

- **平日 11:30〜12:30** (ランチ前)
- **平日 17:30〜20:30** (夜ごはん前)
- **特に夜ごはん需要なら 18:00〜19:30**

## 7. ハッシュタグ戦略

### Instagram (5個まで)

3層ミックス:

- **大ボリューム (1個)**: `#テイクアウト` `#デリバリー`
- **ターゲット絞り込み (2-3個)**: `#一人暮らし` `#会社員ごはん` `#Uber Eats` `#出前館`
- **ブランド集約 (1個)**: `#Nobuグルメ`

### 実例 (1本目)

```text
#テイクアウト #デリバリー #一人暮らし #会社員ごはん #Nobuグルメ
```

(注: `#Nobuグルメ` は自分用ブランドハッシュタグ。シリーズ追跡用)

### キャプション本文に散りばめるキーワード

ハッシュタグが5個に絞られた分、本文中に以下を自然に入れて自然言語処理に拾わせる:

- 料理ジャンル: バーガー / カレー / 麺類 / ピザ / サラダ
- アプリ名: Uber Eats / 出前館 / Wolt (※本文のみ、画像には出さない)
- シーン: 仕事帰り / 在宅勤務 / 一人暮らし / 雨の日
- 評価軸: 口コミ / 評価 / 写真 / 配達時間

## 8. 収益化戦略

### Instagram でのマネタイズパス

| 収益源 | 実現性 | 単価 | 備考 |
|---|---|---|---|
| **ナッシュ・三ツ星ファーム等の冷凍宅配** | ◎ | ¥3,000〜¥5,000/件 | A8.net で承認率高い、グルメと親和性◎ |
| **ふるさと納税 (グルメ返礼品)** | ◎ | ¥1,000〜¥3,000/件 | 楽天ふるさと納税は楽天アフィリエイト対応 |
| **PR案件 (個別タイアップ)** | ◯ | ¥1万〜¥10万円/件 | フォロワー6,000+で実現可能ライン |
| Uber Eats / 出前館 紹介プログラム | ◯ | 紹介ボーナス〜 | プロモコード型・PR表記必須 |
| Wolt 紹介プログラム | ◯ | プロモコード型 | 同上 |
| キッチン家電 | △ | 楽天アフィリ等 | 温め直し系で関連 |
| Instagram ショッピング | △ | - | 食品は限定的 |

### コンプラ表記の必須

- アフィリエイトリンク含む投稿は **「PR」「広告」明記**
- 「絶対お得」「必ず得する」等の断定表現禁止
- 実体験ベースでない事例は **「個人的に」「傾向として」** で柔らかく

## 9. ai-content-studio パイプライン使用方法

「お金の初期設定」と同じ7ステップパイプラインを `takeout-gourmet` チャンネルに対して実行可能:

```bash
# 1. 台本生成
node src/cli.js script takeout-gourmet "テイクアウトグルメ、甘く見てない？"

# 2. ショットプラン生成 (Remotion テーマは takeoutGourmetTheme.ts を使用するよう生成側で指定)
node src/cli.js shot-plan takeout-gourmet <script-path>

# 3. ナレーション整形
node src/cli.js narration takeout-gourmet <script-path>

# 4. VOICEVOX 音声生成 (春日部つむぎプリセット自動適用)
node scripts/generate-genz-audio.js <script.txt> <audio.wav>
# ※ generate-genz-audio.js は voiceRouting[channelId].preset で適切なプリセットを選ぶ

# 5. セグメント計測
node scripts/measure-segments.js <script.txt> --json <segments.json> --final-wav <audio.wav>

# 6. shot plan → scene JSON 変換 (theme: takeoutGourmetTheme を指定)
node src/cli.js remotion-scene takeout-gourmet <shot-plan.json>

# 7. Remotion レンダリング
node src/cli.js remotion-render takeout-gourmet <scenes.json>
```

### 注意点

- `generate-genz-audio.js` (ファイル名は genz-money 由来) はチャンネル汎用で動作する設計
- 将来的にファイル名を `generate-voicevox-audio.js` 等に汎用化するリファクタリングを検討
- Remotion コンポジション (`GenzMoneyShort.tsx`) は将来 channel別に分割するか、theme を props で受け取る形に拡張

## 11. リスク・コンプラ要件

### 法的リスク

| リスク | 対策 |
|---|---|
| 実在店舗の名誉毀損 | 個別店舗名・住所・写真は出さない |
| 商標権侵害 (デリバリーアプリのロゴ) | 公式UIを模さない、アイコンは抽象化 |
| 食品衛生関連の誤情報 | 一次情報 (消費者庁・厚労省等) を参照、断定回避 |
| ステマ規制 (景品表示法) | PR表記必須、消費者庁ガイドライン準拠 |

### ブランドリスク

| リスク | 対策 |
|---|---|
| 既存フォロワーの離脱 (ノウハウ系への抵抗) | 1本目はコンセプト動画、徐々にノウハウ移行 |
| フィードとリールの世界観ギャップ | カバー画像でフォント・配色統一 |
| 攻略アカウント感 | 共感型・生活提案型を定期的に挟む (5本目で再調整) |
