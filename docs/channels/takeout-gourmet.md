## 2. トーン・表現ルール

### 推奨トーン (warm-friendly)

- 実体験ベース
- やさしい・共感ベース
- 保存したくなる実用系
- 友達に送りたくなる
- 「個人的に」「傾向として」「初めて頼む店で」のような柔らかい表現

### 禁止トーン (店舗批判・煽り回避)

- 「地雷店を見抜け」「この店はダメ」「このジャンルは頼むな」
- 「情弱はここで損してる」「Uber で損する人の特徴」
- 「絶対」「必ず」等の強い断定
- 「外食する人は損している」「テイクアウトは手抜きではない」（防御的表現も避ける）

### 推奨フック例

- 「テイクアウトグルメ、甘く見てない？」(1本目専用・少し攻めの引き)
- 「Uber Eats で失敗したくない人、ここ見てください」
- 「デリバリーで麺類を頼んで後悔したことありませんか？」
- 「初めての店でテイクアウトする前に、これだけ見て」
- 「個人的に失敗しにくい選び方」

### 1カット1メッセージ原則

- 「お金の初期設定」と同じく、1カット1メッセージを維持
- ただし冒頭は **情報密度より感情・共感** を優先 (お金の初期設定は数字でフック、グルメは共感でフック)

## 5. 動画テンプレート (4種)

### テンプレ1: 3ポイント解説

- 尺: 25-30秒
- 用途: 店選び・口コミ・写真・配達時間
- 構成: フック → よくある失敗 → 3ポイント → 補足 → 保存CTA

### テンプレ2: 向き・不向き診断

- 尺: 20-25秒
- 用途: 料理ジャンル別
- 構成: 二択提示 → メリット → 注意点 → 推奨注文方法 → コメント誘導

### テンプレ3: あるある対処法

- 尺: 20秒
- 用途: トラブル・共感
- 構成: あるある提示 → なぜ起こるか → 次回の対策 → 保存・共感コメント誘導

### テンプレ4: 比較型

- 尺: 25-30秒
- 用途: バーガー vs カレー、ピザ vs 麺 など
- 構成: 二択提示 → A特徴 → B特徴 → 結論 → どっち派？コメント誘導

### コンセプト型 (1本目専用)

- 尺: 30秒
- 用途: ブランド世界観・期待感醸成
- 構成: フック → 魅力訴求 → 生活感シーン → シリーズ予告

## 9. ai-content-studio パイプライン使用方法

「お金の初期設定」と同じ7ステップパイプラインを `takeout-gourmet` チャンネルに対して実行可能:

```bash
# 1. 台本生成
node src/cli.js script takeout-gourmet "テイクアウトグルメ、甘く見てない？"

# 2. ショットプラン生成 (Remotion テーマは takeoutGourmetTheme.ts を使用するよう生成側で指定)
node src/cli.js shot-plan takeout-gourmet <script-path>

# 3. ナレーション整形
node src/cli.js narration takeout-gourmet <script-path>

# 4. VOICEVOX 音声生成 (春日部つむぎプリセット自動適用)
node scripts/generate-genz-audio.js <script.txt> <audio.wav>
# ※ generate-genz-audio.js は voiceRouting[channelId].preset で適切なプリセットを選ぶ

# 5. セグメント計測
node scripts/measure-segments.js <script.txt> --json <segments.json> --final-wav <audio.wav>

# 6. shot plan → scene JSON 変換 (theme: takeoutGourmetTheme を指定)
node src/cli.js remotion-scene takeout-gourmet <shot-plan.json>

# 7. Remotion レンダリング
node src/cli.js remotion-render takeout-gourmet <scenes.json>
```

### 注意点

- `generate-genz-audio.js` (ファイル名は genz-money 由来) はチャンネル汎用で動作する設計
- 将来的にファイル名を `generate-voicevox-audio.js` 等に汎用化するリファクタリングを検討
- Remotion コンポジション (`GenzMoneyShort.tsx`) は将来 channel別に分割するか、theme を props で受け取る形に拡張

## 12. 品質改善ロードマップ (制約と方針)

### 運用制約 (確定 2026-05-20)

このチャンネルの動画制作は以下の制約下で運用する:

1. **AI自動化最優先** (手作業編集は発生させない・CapCut/Canva 等の手動編集ツールは使わない)
2. **実写素材は原則使わない** (グルメ写真・動画が必要な場合は AI 生成で対応)
3. **AI ツールは固定料金プランのみ** (API 従量課金モデルは使わない)

使えるツール (固定料金 or 無料):

- ✅ Remotion (無料・ローカル)
- ✅ VOICEVOX (無料・ローカル)
- ✅ ChatGPT Plus (固定月額、DALL-E 3/Sora の生成枠を活用)
- ✅ Claude (固定月額)
- ❌ Runway (API従量課金) — 使わない
- ❌ Midjourney (従量) — 使わない
- ❌ ElevenLabs (チャンネル外で別途検討) — 不要
- ❌ CapCut / Canva — 使わない

### Phase 1 (実装済 2026-05-20)

- v1: 暗色パレット・5シーン・24秒 → 「全体的に暗い・地味」フィードバックを受けて撤回
- v2: クリーム+ビビッドパレット・10シーン・37秒・絵文字統合 → 「折り返し・位置ズレ・チープ」フィードバック
- v3 (現在): WarmBulletList の折り返し修正・改行明示化・テキスト密度削減

### 既知の品質限界 (v3 時点)

Instagram のプロ品質グルメリールと比較した**ギャップ**:

| 要素 | プロのリール | v3 現状 | 改善余地 |
|---|---|---|---|
| ビジュアル主体 | 実写の料理写真/動画 | テキスト+標準絵文字 | 大 |
| アイコン | カスタムイラスト/SVG | 標準絵文字 (汎用) | 大 |
| 背景 | 質感ある写真/動画 | フラットグラデーション | 中 |
| タイポグラフィ | 複数フォント・ウェイト | 単一フォント | 中 |
| モーション | キーフレーム作り込み | 標準 spring/fade | 中 |
| 音響 | BGM + SE | 音声のみ | 大 |
| レイアウト | 余白多め・情報少なめ | テキスト密度高め | 小 |

### Phase 2 候補 (Instagram 1本目公開後、データを見て判断)

制約に収まる範囲での品質向上策:

#### 案2-A: カスタム SVG イラストライブラリ (無料・コード生成)

- React コンポーネントとして食べ物/生活雑貨イラストを実装
- 例: スタイライズされたバーガー/ラーメン/紙袋/スマホUI
- 標準絵文字より一段クオリティが上がる
- **コスト**: 開発時間のみ
- **効果**: 大 (絵文字感の解消)

#### 案2-B: AI 生成画像アセット (ChatGPT Plus 内、固定料金)

- DALL-E 3 / Sora で「テイクアウト風景」「料理イメージ」「配達バッグ」等の静止画/動画を生成
- 一度生成して `content/takeout-gourmet/assets/` に保存、再利用前提
- ChatGPT Plus の月間生成枠内で運用 (追加課金なし)
- **コスト**: 生成時間のみ
- **効果**: 大 (実写感の代替)

#### 案2-C: タイポグラフィシステム強化 (無料・コード)

- Google Fonts 追加: Noto Serif JP / M PLUS Rounded / Zen Maru Gothic
- ヘッドラインとサブで別フォント切替
- グラデーションテキスト・文字 1 文字ずつ出現アニメ
- **コスト**: 開発時間のみ
- **効果**: 中

#### 案2-D: BGM + SE 統合 (無料・既存スタック)

- `public/se/` に既存の SE パレット (8役割) あり、takeout-gourmet でも活用可
- YouTube Audio Library から royalty-free BGM をダウンロードして mp3 として組み込み
- VOICEVOX 音声と mixing (Remotion 内で Audio コンポーネント重ねがけ)
- **コスト**: 楽曲選定時間
- **効果**: 大 (動画の「商品感」UP)

#### 案2-E: モーション設計の作り込み (無料・コード)

- 文字を 1 文字ずつ出現させる typewriter アニメ
- パララックス効果 (前景・中景・遠景)
- カスタム easing curve (cubic-bezier)
- パーティクル/紙吹雪エフェクト
- **コスト**: 開発時間 (中〜大)
- **効果**: 中

### Phase 2 の優先順位 (推奨)

データドリブンで判断するが、効果対コストで以下の順序を想定:

1. **案2-D BGM/SE 統合** (効果大・コスト小)
2. **案2-B AI 生成アセット** (効果大・既存ツール活用)
3. **案2-A カスタム SVG イラスト** (効果大・差別化大)
4. **案2-C タイポグラフィ** (効果中・コスト小)
5. **案2-E モーション設計** (効果中・コスト大)

### Phase 1 → Phase 2 判断基準

1本目公開後、24-72時間で以下を観察:

- **視聴維持率 < 40%**: 早急に Phase 2 (案2-A/B 同時着手)
- **視聴維持率 40-60%**: Phase 2 (案2-D から段階導入)
- **視聴維持率 > 60%**: 現状品質で十分、量産優先
