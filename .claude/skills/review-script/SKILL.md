# 台本レビュー・改善

生成済みの台本を読み込み、プロの YouTube スクリプトライターの視点でレビュー・改善します。

## 手順

1. `$ARGUMENTS` で指定された台本ファイルを読む。指定がない場合は `content/` 配下の最新の台本を探す:

```bash
ls -t content/*/scripts/*.md | head -5
```

2. 以下の **6つの観点** でスコアリングする（各10点、合計60点）:

   | 観点 | チェック内容 |
   |------|------------|
   | **フック力** | 冒頭30秒で視聴者を掴めるか。「見なきゃ損」と思わせるか |
   | **構成** | セクションの流れが論理的か。テンポが落ちる箇所がないか |
   | **視聴維持率** | パターンインタラプト（驚き、質問、転換）が2-3分ごとにあるか |
   | **トーン** | チャンネルのターゲット層に合った語り口か |
   | **CTA** | 自然な形で登録・いいねを促しているか。押し売り感がないか |
   | **SEO** | キーワードが自然にスクリプトに含まれているか |

3. スコアと改善ポイントを報告する

4. ユーザーが同意したら、台本ファイルを直接編集して改善を適用する

## 改善時の原則

- **フックは3パターン提案する** — 質問型、衝撃事実型、ストーリー型
- **削るのが最善の編集** — 冗長な箇所を見つけたら短縮する
- **B-ROLL 指示を追加** — 視覚的に退屈な箇所にビジュアル指示を補う
- **[TEXT ON SCREEN] を活用** — 重要な数字や引用はテロップ指示を入れる
